# notedocal 🌙

**Note** + **To-Do** + **Cal**endar — заметки, список дел и календарь в одном приложении,
с поддержкой **григорианского** и **исламского (хиджра)** календарей.

Кроссплатформенное веб-приложение: открывается в любом браузере (телефон, планшет,
компьютер), работает офлайн, данные хранятся прямо на устройстве. Можно «установить»
на телефон как обычное приложение (PWA).

## Возможности

- 📅 **Календарь** — сетка месяца с двойными датами (григорианская + хиджра), сегодняшний
  день и выбранный день подсвечены. На днях с задачами/заметками — цветные точки.
- ✓ **Задачи** — список дел с приоритетами, сроками и отметкой выполнения.
- 📝 **Заметки** — быстрые текстовые заметки с редактированием.
- 🗓 **День** — кликнув по дню в календаре, можно добавить задачи и заметки именно на эту дату.
- 🌗 Светлая и тёмная темы (автоматически по системным настройкам).
- 💾 Данные хранятся в `localStorage` браузера — приложение работает офлайн и
  без регистрации.
- ☁️ **Синхронизация между устройствами** (опционально) — вход через Google,
  и заметки/задачи едины на Windows, Linux и Android. См. раздел
  «[Синхронизация между устройствами](#синхронизация-между-устройствами)».

Хиджра-даты вычисляются встроенным в браузер механизмом `Intl` (календарь
`islamic-umalqura`, Умм аль-Кура) — без внешних библиотек.

## Технологии

Vite · React · TypeScript · Firebase (Auth + Cloud Firestore).

```bash

npm run dev      # запустить в режиме разработки → http://localhost:5173
npm run build    # собрать продакшен-версию в папку dist/
```

## Синхронизация между устройствами

Приложение — это PWA: один и тот же код открывается в браузере на Windows,
Linux и Android. Чтобы данные были **едины** на всех устройствах, подключается
бесплатный Firebase (вход через Google + база Cloud Firestore). Без настройки
приложение продолжает работать как раньше — локально, без входа.

### Как это работает

- Войдя через Google на каждом устройстве, вы получаете единый набор данных.
- Изменения отправляются в облако и приходят на другие устройства почти мгновенно
  (realtime-подписка Firestore).
- Слияние идёт по принципу «последнее изменение побеждает» (по времени правки);
  удаления переносятся через «надгробия», поэтому удалённое на одном устройстве
  исчезает и на других.
- Всё работает офлайн: правки копятся локально и досинхронизируются при сети.

### Настройка

Проект Firebase для этого приложения **уже настроен**, а его публичный веб-конфиг
зашит в [`src/lib/firebaseConfig.ts`](src/lib/firebaseConfig.ts) — поэтому
никаких дополнительных действий для работы синхронизации не требуется: соберите
и задеплойте приложение, нажмите «Войти через Google» — и всё.

Действующие правила Firestore: каждый пользователь видит только свои данные;
официальный календарь наблюдений (`shared/calendar`) читают все, а пишет только
администратор (по email):

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Приватные данные пользователя
    match /users/{uid} {
      allow read, write: if request.auth != null && request.auth.uid == uid;
    }
    // Официальный календарь: читают все (в т.ч. без входа), пишет только админ
    match /shared/{doc} {
      allow read: if true;
      allow write: if request.auth != null
        && request.auth.token.email == 'ismailoffism@gmail.com'
        && request.auth.token.email_verified == true;
    }
  }
}
```

> Ключи веб-конфига Firebase **не секретны** — они всё равно попадают в собранный
> JS, видимый в браузере. Доступ к данным защищают правила Firestore выше и
> список авторизованных доменов в Firebase Authentication, а не сокрытие ключей.

### Источники хиджра-даты и роль администратора

Дата по хиджре для каждого дня выбирается по приоритету источников:

1. **Официальный календарь администратора** (`shared/calendar`) — если включён.
2. **Личные наблюдения пользователя** (`users/{uid}.sightings`).
3. **Расчётный календарь** Умм аль-Кура (`Intl`) — помечается как
   предположительный (знак «≈»).

Администратор определяется по email (`ADMIN_EMAIL` в
[`src/lib/firebase.ts`](src/lib/firebase.ts)). Его наблюдения сохраняются в общий
документ и видны всем — в том числе пользователям без входа. Обычный
пользователь может отключить официальный календарь переключателем и опираться
только на свои наблюдения и расчёт.

### Свой проект Firebase (необязательно)

Если хотите подключить **другой** проект Firebase (например, отдельный для
разработки), не меняя код:

1. Создайте проект на <https://console.firebase.google.com>, добавьте
   Web-приложение, включите провайдер **Google** в Authentication и в
   **Authorized domains** добавьте свой домен и `localhost`.
2. Создайте базу **Firestore** и задайте правила, как выше.
3. Локально: `cp .env.example .env` и впишите ключи нового проекта — переменные
   `VITE_FIREBASE_*` перекрывают значения по умолчанию.
4. Для GitHub Pages: те же ключи добавьте в **Settings → Secrets and variables →
   Actions → вкладка Variables**; workflow деплоя подставит их при сборке.
